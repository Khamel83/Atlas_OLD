AWS:reInvent
------------

From SiliconANGLE ( https://siliconangle.com/2023/11/28/amazon-enhances-serverless-data-platforms-kick-off-reinvent-2023/ ) :

Cloud computing giant Amazon Web Services Inc. kicked off its annual re:Invent 2023 conference this evening with a keynote address where it announced a trio of new capabilities that should make it easier for customers to use its serverless database offerings.

The headline announcement was the introduction of Amazon Aurora Limitless Database, which is a new feature for the Amazon Aurora database that can automatically scale beyond its original write limits, making it much easier for developers to scale their applications. Also announced were new capabilities for Amazon ElastiCache and the Amazon Redshift Serverless database, designed to drive up workload performance while optimizing the platforms to manage costs.

I actually thought that this keynote from AWS SVP of Utility Computing Peter DeSantis ( https://www.youtube.com/watch?v=pJG6nmR7XxI ) was really enjoyable: it wasn’t a typical enterprise keynote with logos and awkward executive interviews, but rather a really good explainer of cloud computing generally and serverless specifically. To summarize briefly:

* It used to be that an enterprise had to buy a server, host it in a physical space, provide networking, install software, and only then run their application.
* The initial wave of cloud computing took away the hardware parts of the equation: now, instead of buying a server, you could rent one, complete with hardware and networking. Then, you could install software like an operating system and the various frameworks you needed, and then run your application.
* Serverless removes the need to maintain a server at all (thus the name); you don’t need to install an operating system and keep it up-to-date. You simply run your application using the appropriate APIs for the serverless resources like compute and storage, and they are provisioned to your application on demand.

DeSantis started out by noting that serverless is the best possible manifestation of the promise of cloud computing:

( https://videopress.com/embed/YyjJgLQy?hd=1&cover=1&loop=0&autoPlay=0&permalink=1&muted=0&controls=1&playsinline=0&useAverageColor=0&preloadContent=metadata )

These are the six most important attributes of a cloud computing service. These are the things that we spend a ton of time putting into all of our services. But serverless services let us take these things a step further. For example, serverless is more elastic, because we run over our vast infrastructure and share our capacity across a large number of customers, each customer workload represents a small fraction of the capacity. And serverless is more cost-effective. Unlike a service where you have to pay for what you provision, with a serverless capability you only pay for what you use. And serverless computing allows us to run our infrastructure more efficiently, and that means better sustainability, because the most efficient power is the power you don’t use. And because these services are built from the ground up to run on AWS’s infrastructure, they deliver better security and better availability, taking advantage of native capabilities like AWS Nitro and our availability zone architecture.

The three announcements DeSantis made were about expanded serverless offerings for relational databases, caching, and data warehouses (and to be fair, not everything was truly serverless; the caching offering, for example, has a minimum usage size which means you’re paying a minimum fee regardless). There was a neat digression in the discussion about the database offering (Aurora Limitless) that described how AWS had to solve the problem of ensuring that servers anywhere in the world had the same time down to the millisecond, so that timestamps could be used to keep events in sync; it was also interesting how the company is using machine learning to predict the resources needed for data warehouse queries to provision capacity for more complicated requests.

--------------------------------

You can see it everywhere, though:

* When it comes to video, in the most extreme example there are YouTube creators on one side and YouTube on the other. Even in the case of Hollywood, though, there is opportunity for show runners on one side and Netflix on the other, and no obvious need for networks in the middle.
* When it comes to sports all of the profit is flowing to leagues on one side, and the company potentially the best positioned is Google on the other; to the extent Disney rebuilds ESPN’s value it will be by using it as an anchor to build a broad-based advertising business ( https://stratechery.com/2023/disney-earnings-disney-3-0-streaming-and-sports/?access_token=eyJhbGciOiJSUzI1NiIsImtpZCI6InN0cmF0ZWNoZXJ5LnBhc3Nwb3J0Lm9ubGluZSIsInR5cCI6IkpXVCJ9.eyJhdWQiOiJzdHJhdGVjaGVyeS5wYXNzcG9ydC5vbmxpbmUiLCJhenAiOiJIS0xjUzREd1Nod1AyWURLYmZQV00xIiwiZW50Ijp7InVyaSI6WyJodHRwczovL3N0cmF0ZWNoZXJ5LmNvbS8yMDIzL2Rpc25leS1lYXJuaW5ncy1kaXNuZXktMy0wLXN0cmVhbWluZy1hbmQtc3BvcnRzLyJdfSwiZXhwIjoxNzAzNzc4MTA5LCJpYXQiOjE3MDExODYxMDksImlzcyI6Imh0dHBzOi8vYXBpLnBhc3Nwb3J0Lm9ubGluZS9vYXV0aCIsInNjb3BlIjoiZmVlZDpyZWFkIGFydGljbGU6cmVhZCBhc3NldDpyZWFkIGNhdGVnb3J5OnJlYWQgZW50aXRsZW1lbnRzIiwic3ViIjoiUnNldVlFOGpnSGVQcFdKZjFmQWI0QyIsInVzZSI6ImFjY2VzcyJ9.kIkqwmnswF8PKfqHtzWyDgOk4nXe4qWaZk7Fp8g2bHEuDFCyK1nDDxt-14Jhr6pA-N5ohPuSXKVdOVIB74IxpJLRJQ0vBGhBCfzF1yKqlQcpnK1X8F4f2XPVHWCaWXeGhz47NwTAmUIFAAesJ2bDR7a4bFmdR72eeVuwJw4o9S7_HW92CqC_GRuySmdwSMHW3P9Vg36XLCLTTZMPo39bXjrCthfJEkg4-XWqjboLcXo1D1uRU0u3trouj8E31cOZkd8OnuMpvqEWY2cQfufU9lAc0OYqwzPIijnviTGjekJfDMHspvKbzeemLcM3vC4CVT_bdNn1iIW-XjKF8MBQlw ) on the right.
* When it comes to e-commerce — which is increasingly to say commerce generally — there are niche merchants on one side and massive platforms like Amazon or Shopify on the other; traditional retailers are squeezed in the middle.

In every case the nature of Internet economics — zero marginal costs and zero transactional costs — make it viable for massive entities that benefit from scale to service even the smallest of customers with the same capabilities that, in the analog world, were only available to intermediary entities like newspapers, networks, or retailers. Moreover, those capabilities are available at exceptionally low prices, particularly when you are just getting started: you can set up a blog, or upload a video to YouTube, or set up a Shopify shop basically for free. This does, to be fair, increase the amount of bad writing or bad videos or cheap junk in the world, but it also provides the possibility for niche-focused and truly differentiated offerings to build viable businesses that reach the entire world.

So it is with software and serverless platforms like AWS. A single developer can basically run a web application directly on AWS, with everything in the middle, from hardware to operating system, completely abstracted away. AWS and other public clouds are providing the technical manifestation of a smiling curve.

That’s not to say that servers are going away: DeSantis himself noted that there is a lot of legacy code that presumes the presence of an always-on server, and of course AWS will continue to support that. What he didn’t say, for understandable reasons, is there are also applications for which it makes much more economic sense to run an always-on server (like Prime Video ( https://www.primevideotech.com/video-streaming/scaling-up-the-prime-video-audio-video-monitoring-service-and-reducing-costs-by-90 ) ): a dedicated server on AWS is cheaper on a minute-by-minute basis than serverless; the savings from the latter come from it running less of the time, having far simpler scalability capabilities, and not needing to have dedicated developer operations teams. And, I would add, there are companies for which it makes sense to run their own dedicated datacenters, including hardware, which is cheaper still on a minute-by-minute basis, albeit with even more complexity and even less flexibility.

None of that, though, changes the fact that it is serverless that is “native” to the inherent structure of the Internet. The Internet, in industry after industry, favors the massive provider benefiting from scale, and the highly differentiated leveraging the cost structures that said scale makes possible: everything in the middle is legacy.

Amazon Web Services Inc. today announced a range of new artificial intelligence service capabilities that are enhanced using foundation models. Announced at the annual re:Invent 2023 conference, the enhanced capabilities include Amazon Transcribe now offering FM-powered language support and AI-enhanced call analytics, Amazon Personalize now using FM to generate more compiling content, and Amazon Lex now using large language models to provide accurate and conversation responses.

There were no major new AI announcements from AWS (although the conference is still going on), but it is worth noting that a lot of actual AI use cases will look like this. To put it in the same framework I used for servers:

* Some small number of companies will have their own foundation models; this is like owning your own data center.
* A larger number of companies will fine-tune foundation models for their own use cases, likely using a service like AWS Bedrock; this is like running your own server on a cloud provider.
* A huge number of applications will benefit from AI by incorporating specific capabilities via APIs available from their cloud provider; this is analogous to serverless.

These announcements fall in that last bucket: now an application on AWS can have foundation-model driven translation, recommendation, or even chat responses, without needing to have their own foundation model or do extensive fine-tuning. Of course all of this depends on the underlying foundation models being good, but, as AWS wants to remind you, it also depends on the underlying foundation models being reliable over the long run. To that end, this comment from AWS CEO Adam Selipsky to SiliconANGLE about the OpenAI kerfuffle ( https://siliconangle.com/2023/11/27/reinvent-exclusive-aws-ceo-adam-selipsky-reveal-new-generative-ai-stack/ ) was exactly what you would expect:

We have been saying since we started laying out our generative AI strategy, almost a year ago, how important it is for customers to have choice. Customers need multiple models – there is no one model and no one model provider to rule them all. The events of the last few days have shown the validity and the usefulness behind the strategy that AWS has chosen. Customers need models they can depend on. Customers need model providers that are going to be dependable business partners. And customers need cloud providers who offer choice and offer the technology that enables choice. Those cloud providers need to want to offer choice as opposed to being driven to provide choice. The importance of these attributes, which AWS has been talking about and acting on for more than 17 years, has become abundantly clear to customers in relation to generative AI.

It’s a fair argument!
